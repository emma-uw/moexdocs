#### TCodMode - COD-подписка (длина: 1, тип: ekGroup)

Тип подписки на COD-режим. COD-режим - Cancel on Disconnect - режим снятия заявок при потере соединения с ТС или при отключении от ТС. Доступные режимы: "Не используется", "Всегда" и "По требованию"

| Константа | Значение | Описание |
| --- | --- | --- |
| N | Не используется | Функционал COD не доступен |
| A | Всегда | COD-режим включен постоянно |
| O | По требованию | COD-режим может быть включен по требованию |
| W | Технический режим 1 | COD-режим включен постоянно, при неактивности участник отключается от ТС |
| C | Технический режим 2 | COD-режим включен постоянно, при неактивности участник отключается от ТС без снятия заявок |

#### TImmCancel - Условие исполнения (длина: 1, тип: ekGroup)

Перечень возможных условий исполнения заявки:

"Поставить в очередь" - для лимитной заявки неисполненный остаток ставится в очередь с курсом, указанным в заявке, для рыночной заявки неисполненный остаток снимается с торгов;

"Полностью или отклонить" - лимитная или рыночная заявка исполняется только в том случае, когда заявка может быть удовлетворена полностью;

"Снять остаток" - для лимитной заявки неисполненный остаток снимается с торгов, для рыночной заявки не используется.

| N | Н | Полностью или отклонить |
